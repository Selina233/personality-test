# 问卷版本管理

本目录用于存储和管理所有版本的问卷配置文件。

## 版本命名规范

- 格式：`vX.Y.Z.json`
- X: 主版本号（问卷结构重大变更）
- Y: 次版本号（增加或删除题目）
- Z: 修订号（修改题目文案、修复错误）

## 目录结构

```
questionnaire-versions/
├── README.md           # 本文件
├── v1.0.0.json        # 测试版本（2+5题）
├── v2.0.0.json        # 完整版本（30+60题）- 待发布
└── version-index.json  # 版本索引文件
```

## 版本文件格式

每个版本文件包含：
- `version`: 版本号
- `releaseDate`: 发布日期
- `description`: 版本描述
- `status`: 版本状态（testing/stable/deprecated）
- `questionCounts`: 题目数量统计
- `questions`: 完整题目列表
  - `behavior`: 行为测试题
  - `bfi2`: BFI-2人格测试题
- `changelog`: 变更日志
- `notes`: 备注说明

## 使用说明

1. **创建新版本**：复制最新版本文件，修改版本号和内容
2. **更新索引**：在 `version-index.json` 中添加新版本记录
3. **更新根目录配置**：修改根目录的 `questionnaire-version.json` 指向当前生效版本
4. **数据追踪**：用户提交的数据会自动记录填写时的版本号

## 注意事项

- 不要删除已发布的版本文件
- 每次题目修改都应创建新版本
- 保持版本号递增
- 及时更新 changelog
